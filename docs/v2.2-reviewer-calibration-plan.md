# v2.2 — Reviewer Calibration Plan

**Plan only. No tools scored.** Operationalises §05C ("Reviewer calibration") so that
"every score is reproducible" is true in practice, and so the **provisional** tolerances
(±0.5 pillar / ±0.3 composite) can be confirmed or revised before any score goes live.
Source: `~/Documents/methodology`.

## Purpose
A rubric in prose is read differently by different people. Calibration is what holds the line:
every reviewer scores the same fixed set, we measure the spread, and we only trust (and publish)
scores once inter-rater agreement is inside an evidenced tolerance.

## 2. Roles & separation of duties (§05C)
- Minimum **two reviewers** per calibration round (a primary and a second reader).
- A reviewer who holds any partnership/outreach relationship with a vendor (the "Oliver" side)
  **may not score that vendor**. **Donna enforces** this conflict check.
- Each score records: **methodology version · verified date · reviewer initials · review-basis
  tier** (Desk / Hands-On / Classroom).

## 3. Scoring procedure (every round)
1. **Blind pillar scoring** — score each pillar on its own evidence with the running composite
   **hidden** (defeats the halo effect). Tooling must not reveal the composite until all five
   pillars are entered.
2. **0.5 steps**, placed against the §05B band descriptor + the **anchor exemplar** for that band
   (see the anchor proposal), with ≥1 cited evidence item per pillar.
3. Apply order of operations: weighted sum → graduated cap → half-up round → verdict band.
4. **Second-reader mandatory** when: the verdict is **Avoid**, or the composite sits **within
   0.2 of a band boundary**. (In batch 1 this already catches Google Classroom 7.95 and
   ClassDojo 7.85.)
5. Record everything to the review file / KCSIE Audit Trail Receipt.

## 4. Tolerance-validation experiment (confirm the provisional ±0.5 / ±0.3)
The ±0.5 pillar / ±0.3 composite figures are **provisional and must not be published as the
standard until tested against double-scored real reviews** (§05C). Method:
1. Have **≥3 reviewers independently, blind-score the full calibration set** (slots 1–10).
2. For each tool, compute the **inter-rater spread**: max−min per pillar, and per composite.
3. Summarise across the set: e.g. the **95th-percentile** pillar spread and composite spread.
4. **Set the published tolerance at (or just above) that 95th percentile** — i.e. the tolerance
   should reflect the agreement we can actually achieve, not an aspiration.
5. If the observed spread is **wider** than ±0.5 / ±0.3, do **not** loosen the tolerance — instead
   tighten the rubric/anchors (add band text, sharpen boundaries) and **re-run** until the spread
   fits. If it's comfortably **inside**, confirm the figures.
6. Only after this passes are the tolerances "confirmed" and scores eligible to go live.

**Acceptance criterion to publish ANY live score:** the calibration set is scored by ≥3
reviewers within the (now-confirmed) tolerance, AND each published tool has a named reviewer +
cited evidence + basis tier + second-reader where required.

## 5. Cadence & ownership
- **Onboarding:** every new reviewer scores the full calibration set before scoring live tools;
  must land within tolerance against the reference scores.
- **Quarterly:** all reviewers re-score the set; drift outside tolerance triggers re-training and
  a rubric/anchor review. Re-confirm anchors at the same time.
- **Owner:** Donna runs the conflict check + words-to-avoid + records; a named human reviewer
  owns the editorial scores. (AI drafts — like calibration batch 1 — are inputs, never the
  named reviewer.)
