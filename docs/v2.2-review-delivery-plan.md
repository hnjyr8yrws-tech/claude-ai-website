## The factory (how each tool is processed)
One desk-review agent per tool, applying the **five specialist rubrics** (Safeguarding · Data
Privacy · Age Suitability · Accessibility · Transparency) in a single research pass, then the
workflow computes the score deterministically:
- Agent gathers **public evidence** per pillar (privacy policy, DPA, trust/security, terms, age
  policy, accessibility statement/VPAT, safeguarding notes, model/transparency) — verified only,
  never invented; un-findable items recorded as **gaps** and lower the confidence.
- v2.2 maths applied in code: Desk ceiling 8.5 on S & P → weighted (35/25/15/15/10) → graduated
  cap `min(S,P)+1` → half-up round → verdict band → Avoid trigger (S or P < 4.5).
- **Staff-only systems** (MIS / safeguarding / monitoring) use the v2.2.1 Safeguarding
  interpretation (Class A vs B).
- Output per tool: pillar scores · composite · verdict · confidence · evidence summary ·
  evidence gaps · review basis.

> **Five specialists, consolidated per tool.** A literal 5-agents-per-tool design would be
> 1,210 agents — over the workflow's 1,000-agent cap. So the five pillar *rubrics* run inside one
> agent per tool. Same coverage, fits the cap, and far faster — which is what hitting Sunday needs.

## Flags (only these are surfaced; everything else flows through)
- **Avoid verdict** (S or P < 4.5, or composite < 6.0)
- **Low confidence** (key documents blocked/missing)
- **Missing evidence** (any pillar gap, or a legal-barrier pillar with no evidence → not-scoreable)
