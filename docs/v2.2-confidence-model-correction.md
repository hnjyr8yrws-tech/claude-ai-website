# v2.2 — Confidence-Model Correction

**Founder correction. Pauses the factory. Two dimensions are now separated.** No live scores, UI
or deployment changes. Source: `~/Documents/methodology`.

## The error being corrected
The factory treated **absent documentation as a failing score** — a strong, UK-hosted,
KCSIE-2025-aligned safeguarding tool (Securus) was driven to *Avoid* because it had no published
VPAT or model disclosure. That conflates *"we couldn't verify it"* with *"it is unsafe."* They are
not the same.

## The model (UK procurement reality)
- Schools must **assess risk** and **protect children and pupil data** — that is the school's duty.
- **Ofsted does not evaluate individual AI tools.**
- **Missing documentation is a *procurement risk*, not proof a product is unsafe.**

So a Promptly review now reports **two independent things:**

### 1. Promptly Score — *best evidence-based assessment of school suitability*
Driven by **verified** findings. A real, verified safeguarding / privacy / product issue lowers
the score. **Absence of documentation does not, by itself, lower the score.**

### 2. Evidence Confidence — *how confident we are in that assessment*
**High / Medium / Low.** Driven by how much was verified. Missing/blocked/gated documentation
lowers **Evidence Confidence first**, and is listed explicitly — it does not silently sink the score.

## When to use Avoid (verified only)
**Avoid is reserved for a *verified* disqualifier:**
- verified **safeguarding** concern, or
- verified **privacy** concern, or
- product **retired / unavailable**, or
- **terms clearly unsuitable** for school use (e.g. permits selling pupil data for ads; vendor
  insists on being controller of pupil data; minimum-age terms that contradict the promoted use), or
- tool **clearly unsuitable for the promoted educational use**.

**If documentation is missing but no direct risk is verified:**
- do **not** auto-Avoid;
- use **Proceed Carefully** or **Worth Considering** as appropriate;
- set **Evidence Confidence = Low**;
- **list the missing evidence** clearly.

## Factory logic update (to implement before the next batch)
Per pillar, the agent now returns **score + `evidenceStatus` (verified / partial / absent) +
`verifiedConcern` (true only for a real, verified problem)**. Plus, per tool: `audience`
(pupil-facing / staff-only / staff-or-adult-wellbeing), an explicit `avoidGround`
(none / verified-safeguarding / verified-privacy / retired / terms-unsuitable / unsuitable-for-use),
and an overall `evidenceConfidence`.

Scoring rules:
1. **Absent + no verified concern:**
   - *Quality pillar* (Age / Accessibility / Transparency) → mark **N/A**, rescale weights — do **not** score it low for absence.
   - *Legal-barrier pillar* (Safeguarding / Privacy) with **no evidence at all** → **NOT-SCOREABLE** (never fabricated), Evidence Confidence None.
2. **Verified concern** on any pillar → the agent's (lower) score **does** apply.
3. **Avoid** fires **only** when `avoidGround` ≠ none (a verified disqualifier), or a legal-barrier
   pillar has `verifiedConcern` **and** a verified failing score (< 4.5). A low composite caused by
   *absent* pillars can no longer trigger Avoid.
4. **Audience framing:** assess each pillar for the tool's actual audience. A staff / adult
   wellbeing tool is **not** judged against pupil Age-suitability or pupil-safeguarding as if it were
   pupil-facing; pupil-specific pillars become N/A (or the mismatch becomes `unsuitable-for-use`
   *only if it is promoted to pupils*).
5. **Evidence Confidence** is an output in its own right (High/Medium/Low), with the missing items
   listed — separate from the score.

Everything else (weights 35/25/15/15/10, desk ceiling 8.5, graduated cap, half-up rounding,
verdict bands, second-reader on Avoid/band-edge) is unchanged. The updated factory script is staged;
**no tools will be processed until you approve this correction.**
