# v2.2 — Anchor Exemplar Proposal

**Proposal only. No new tools scored.** §05B requires each pillar band to carry a named
*anchor exemplar* ("one per band where one genuinely fits") so every reviewer places a finding
against the same reference. Today they are all `[ANCHOR — tool name]` placeholders. This
proposes **candidate** anchors from the five calibration tools + the 25-tool evidence audit,
and — importantly — flags the bands we **cannot yet anchor**.

## Rules for an anchor (from §05B/§05C)
- One named, currently-reviewed tool per band, per pillar.
- Should ideally be **Hands-On / Classroom** basis — a desk anchor is weaker (it can't sit in
  the 9–10 band of Safeguarding/Privacy at all, per the 8.5 desk ceiling).
- Anchors are re-confirmed at the quarterly calibration (see the calibration plan).

> ⚠️ **Coverage caveat.** We have only **one hands-on tool (MagicSchool)** and four desk tools,
> all scoring mid-to-high. So we can propose **upper-band** anchors only; the **5–6, 3–4, 0–2**
> bands have **no candidate** yet and need tools we have not scored (including some that will
> score badly). Those gaps are listed explicitly — do not publish a rubric with empty low bands.

## Summary of gaps (must fill before publishing the rubric)
- **No anchors below 5–6 on any pillar**, and **no Avoid-tier (3–4, 0–2) anchors at all.** A
  calibration rubric with empty low bands trains reviewers only on "good" tools — they'll have
  no reference for failing ones.
- **Privacy & Accessibility 9–10** unanchored (desk ceiling / no verified-AA tool).
- Several candidates are **desk-basis**, so they are *provisional* anchors — replace with
  hands-on/classroom exemplars where possible.
