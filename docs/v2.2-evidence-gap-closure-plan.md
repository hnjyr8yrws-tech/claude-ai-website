Turns the readiness report's gaps into an actionable closure worklist for the 16 "missing
evidence" tools (+ the 2 "not currently scoreable"). **No scoring.** Goal: get each tool to the
point where all five pillars have a cited, verified evidence item (or a legitimate N/A) at the

## Cross-cutting notes
- **Accessibility is the most common gap** — batch a VPAT request to all vendors that have one
  request-only, and plan hands-on WCAG 2.2 AA audits for those with none.
- **Assistive/non-generative tools** (Texthelp, Immersive Reader, Sparx) still need an explicit
  **Safeguarding stance** under v2.2.1 — Safeguarding can't be N/A even when the harm surface is
  low.
- **Hands-on testing** doubles as evidence closure *and* the only route past the 8.5 desk ceiling
  — prioritise it for the high-visibility tools (the Homepage set).

## Definition of done (per tool)
A tool exits "missing evidence" when **all five pillars** carry a cited, verified evidence item
(or a legitimate Age/Transparency/Accessibility N/A) at a recorded basis tier, with the
blocked-page and contradiction items resolved. Then — and only on CR's go-ahead — it becomes
eligible for scoring.
