## Ruling (Decision 1 — Option 3 adopted)
- **Safeguarding remains a universal pillar.** No N/A. No replacement pillar. **One Promptly
  Score standard** for every tool.
- For **staff-only systems** (no pupil-facing surface), Safeguarding is **interpreted** as:
  > *"The extent to which the system supports, protects or could undermine safeguarding practice
  > within a school environment."*
- This replaces the literal "child-harm exposure" reading **only for the staff-system class** —
  the pillar, its 35% weight, and "never N/A" all stand.

## Classifying a tool
A tool is a **Staff-Only System** when pupils never hold an account / never interact with it; it
is operated by staff. Record the classification in the review. Two sub-classes:

### Class A — Admin / MIS / workflow systems
*(e.g. SIMS, Bromcom, Arbor, ScholarPack, Pupil Asset, Satchel One, FFT Aspire, SISRA, 4Matrix,
Compass, Schoolytics)*
Safeguarding assesses how the system **supports and protects** safeguarding practice:
- **Permissions** — role-based access; least-privilege; control over who sees sensitive data.
- **Audit trails** — who did what, when; tamper-evidence.
- **Reporting integrity** — accuracy/reliability of the records and reports staff rely on.
- **Safeguarding workflow support** — does it support (or get in the way of) proper safeguarding
  process?

### Class B — Dedicated safeguarding systems
*(e.g. CPOMS, MyConcern, Smoothwall Monitor, Impero, Securus, Lightspeed)*
Safeguarding assesses **how well the system performs its safeguarding function**:
- **Safeguarding effectiveness** — does it reliably do the safeguarding job it claims?
- **Case management** — recording, linking, oversight of concerns.
- **Escalation** — routing/alerting to the right people in time.
- **Monitoring** — (for filtering/monitoring tools) detection quality, coverage, false-positive
  handling.
- **Record integrity** — security and trustworthiness of safeguarding records.

## Suggested band guidance (for CR review — operationalises the bands)
The 0.5-step scale and the §05B band *shape* are unchanged; this restates them for the staff-
system interpretation. (Desk-Review ceiling of 8.5 still applies.)

| Band | Class A (admin/MIS) | Class B (dedicated safeguarding) |
|---|---|---|
| 9–10 | Granular least-privilege permissions, complete tamper-evident audit trail, demonstrably reliable reporting, actively supports safeguarding workflow. | Demonstrably effective safeguarding function: robust case management, reliable escalation/alerting, strong monitoring, tamper-evident records. |
| 7–8 | Sound permissions & audit, reporting reliable, supports workflow with minor gaps. | Effective with minor gaps; case management/escalation present and working; record integrity sound. |
| 5–6 | Coarse permissions or weak audit; reporting or workflow support has material gaps. | Safeguarding function works but with material gaps (weak escalation, thin audit, gaps in coverage). *(Capped band.)* |
| 3–4 | Poor access control / no meaningful audit / unreliable records — could **undermine** safeguarding practice. | Safeguarding function unreliable or easily defeated — could undermine protection. *(Triggers Avoid.)* |
| 0–2 | Actively unsafe handling of safeguarding-relevant data/process. | A safeguarding tool that fails at safeguarding. *(Avoid.)* |

## Exclusive-boundary clarification (protects the §05B moat)
- **Data Privacy (unchanged):** data residency, retention, deletion, DPA terms, training-use,
  sub-processors, ICO Children's Code. *"Where the data goes."*
- **Safeguarding (this note):** permissions/audit/workflow (Class A) and safeguarding-function
  efficacy + record integrity (Class B). *"Whether the system protects or undermines
  safeguarding practice."*
- A records breach has both a Privacy aspect (how the data was exposed → Privacy) and a
  Safeguarding aspect (integrity/trustworthiness of safeguarding records → Safeguarding). Score
  the **specific finding** in one pillar per the cause; cross-reference, do not double-count.
